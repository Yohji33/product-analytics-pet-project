# Инструкция по сборке дашборда в Power BI

## Цель

Собрать первый дашборд по e-commerce проекту и научиться переводить SQL/Python-аналитику в понятную визуальную витрину.

Дашборд должен отвечать на вопросы:

- какая общая выручка
- сколько заказов и пользователей
- как меняется выручка по месяцам
- где пользователи отваливаются в воронке
- какие каналы и источники работают лучше
- какие каналы окупаются с учетом маркетинговых расходов
- какие категории товаров дают больше выручки
- на каких устройствах выше конверсия

## Почему начинаем с CSV

В реальной работе Power BI часто подключают напрямую к базе данных.
Но для первого обучения удобнее начать с подготовленных CSV-файлов:

- меньше технических ошибок
- проще понять визуализации
- легче повторить проект на другом компьютере

После этого можно перейти к подключению Power BI напрямую к PostgreSQL.

## Подготовленные данные

Файлы лежат в папке:

```text
dashboards/powerbi_data/
```

Основные датасеты:

- `kpi_overview.csv` - общие KPI
- `monthly_metrics.csv` - динамика по месяцам
- `funnel.csv` - продуктовая воронка
- `traffic_source_funnel.csv` - воронка по источникам трафика
- `acquisition_channel_metrics.csv` - метрики по каналам привлечения
- `marketing_efficiency.csv` - CAC, ROMI и окупаемость каналов
- `device_metrics.csv` - метрики по устройствам
- `category_revenue.csv` - выручка по категориям
- `top_products.csv` - топ товаров
- `retention_cohorts.csv` - когортный retention

## Шаг 1. Открыть Power BI Desktop

Откройте Power BI Desktop через меню Пуск.

## Шаг 2. Загрузить CSV-файлы

1. Нажмите `Get data`.
2. Выберите `Text/CSV`.
3. Откройте файл `monthly_metrics.csv`.
4. Нажмите `Load`.
5. Повторите для остальных файлов из `dashboards/powerbi_data/`.

Для первого дашборда достаточно загрузить:

- `monthly_metrics.csv`
- `funnel.csv`
- `acquisition_channel_metrics.csv`
- `marketing_efficiency.csv`
- `device_metrics.csv`
- `category_revenue.csv`
- `top_products.csv`

## Шаг 3. Проверить типы данных

В Power BI откройте `Data view` и проверьте:

- `month` должен быть датой
- `revenue` должен быть числом
- `session_to_order_conversion` должен быть decimal number
- `paid_orders_count`, `sessions_count`, `users_count` должны быть целыми числами

Если тип определился неверно, его можно поменять на вкладке `Column tools`.

## Шаг 4. Создать страницу Overview

Создайте первую страницу и назовите ее:

```text
Overview
```

Добавьте карточки:

- `revenue`
- `paid_orders_count`
- `average_order_value`
- `session_to_order_conversion`

Источник для карточек:

- `monthly_metrics.csv`

Для карточек используйте агрегирование:

- Revenue: Sum
- Paid orders: Sum
- Average order value: Average
- Conversion: Average

## Шаг 5. Построить динамику выручки

Визуализация:

- `Line chart`

Поля:

- X-axis: `month`
- Y-axis: `revenue`

Смысл:

Показывает, как менялась выручка по месяцам.

## Шаг 6. Построить динамику заказов

Визуализация:

- `Column chart`

Поля:

- X-axis: `month`
- Y-axis: `paid_orders_count`

Смысл:

Показывает, в какие месяцы было больше оплаченных заказов.

## Шаг 7. Построить воронку

Визуализация:

- `Funnel`

Поля:

- Category: `event_type`
- Values: `users_count`

Источник:

- `funnel.csv`

Важно:

Отсортируйте шаги по `step_order`, чтобы порядок был:

1. `view_item`
2. `add_to_cart`
3. `purchase`

## Шаг 8. Построить выручку по каналам привлечения

Визуализация:

- `Bar chart`

Поля:

- Y-axis: `acquisition_channel`
- X-axis: `revenue`

Источник:

- `acquisition_channel_metrics.csv`

Смысл:

Показывает, какие каналы дают больше денег.

## Шаг 9. Построить окупаемость маркетинга

Визуализация:

- `Bar chart`

Поля:

- Y-axis: `acquisition_channel`
- X-axis: `romi`

Источник:

- `marketing_efficiency.csv`

Смысл:

Показывает, какие каналы окупаются лучше с учетом расходов на маркетинг.

Дополнительно можно построить второй bar chart:

- Y-axis: `acquisition_channel`
- X-axis: `cac`

Этот график показывает, сколько в среднем стоит один платящий покупатель в каждом канале.

## Шаг 10. Построить выручку по категориям

Визуализация:

- `Bar chart`

Поля:

- Y-axis: `category`
- X-axis: `revenue`

Источник:

- `category_revenue.csv`

Смысл:

Показывает, какие категории являются драйверами выручки.

## Шаг 11. Построить конверсию по устройствам

Визуализация:

- `Column chart`

Поля:

- X-axis: `device_type`
- Y-axis: `session_to_order_conversion`

Источник:

- `device_metrics.csv`

Смысл:

Показывает, где выше конверсия: desktop, mobile или tablet.

## Рекомендуемая структура дашборда

Первая страница:

- KPI cards сверху
- график выручки по месяцам
- график заказов по месяцам
- воронка
- выручка по каналам
- ROMI и CAC по каналам
- выручка по категориям
- конверсия по устройствам

Вторая страница, позже:

- retention
- топ товаров
- traffic source funnel
- детализация по каналам

## Следующий уровень

После первого CSV-дашборда можно подключить Power BI напрямую к PostgreSQL.

Для этого нужно использовать views из файла:

```text
sql/06_powerbi_views.sql
```

Тогда Power BI будет получать уже подготовленные витрины из базы данных.
